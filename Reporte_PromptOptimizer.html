Tesis: el flujo "selecciona → clic derecho → optimiza/reemplaza", el funcionamiento sin configuración y el enfoque en el mercado hispano son diferenciadores reales frente a una competencia mayoritariamente atada a la interfaz de ChatGPT o a plataformas web de pago.

Diferenciadores principales

  • Funciona en cualquier web vía clic derecho, no solo dentro de ChatGPT.
  • Reemplazar in situ: sustituye el texto seleccionado sin copiar/pegar.
  • Multi-proveedor con opciones gratis (Pollinations, Gemini, Groq, OpenRouter): el usuario no queda encerrado.
  • Prompt en inglés con respuesta en tu idioma: ángulo útil y poco explotado en el mercado hispano.
  • Privacidad (BYOK): la clave se guarda local, sin servidor intermedio.

La extensión

Funcionalidades implementadas (Fase 1 / MVP, versión 0.4.0).

Optimizar por clic derecho

Selecciona texto → menú contextual → “Optimizar como prompt”. Resultado en un panel flotante aislado (Shadow DOM).

Reemplazar in situ

Sustituye el texto seleccionado por el prompt optimizado en campos editables (usa el setter nativo para que React lo detecte). Se oculta en páginas de solo lectura.

Copia automática

El resultado se copia al portapapeles y ofrece botón de copiar y vista del texto original.

Prompt en inglés, respuesta en tu idioma

Configurable: genera el prompt en inglés (mejor rendimiento) e instruye al modelo a responder en el idioma del texto original.

Cross-browser

Un solo código para Firefox y Chrome/Edge mediante la clave dual de background (service_worker + scripts) y un shim de API.

Ajustes simples

Proveedor, API key (por proveedor), modelo, estilo (claridad / técnico / creativo) e idioma del prompt.

Fix aplicado: Google retiró los modelos Gemini 1.5 y 2.0 (error “model not found for generateContent”). Se actualizó a los modelos 2.5 y se añadió una salvaguarda: si un modelo guardado deja de existir, la extensión cae al modelo por defecto en lugar de fallar.
Por qué no se embebe una clave gratis compartida: el código de una extensión se descarga completo a cada usuario, así que la clave sería extraíble; su cupo gratuito se compartiría entre todos (se agota al instante), cualquiera podría abusarla y el proveedor la baneara — además viola los términos. La alternativa robusta es un proxy propio (ver Monetización, Camino B).

Detalle por competidor

Promptimize AI — competidor más directo

Extensión de navegador con botón “enhance” que transforma prompts. Funciona con ChatGPT, Claude y Gemini.

  • Precio: gratis 10 mejoras/día; Pro $12/mes.
  • Debilidad explotable: atada a las interfaces de chat; sin “responder en tu idioma”; un proveedor por uso.
AIPRM — librería de plantillas

Más de 4.500 plantillas creadas por la comunidad, centrado en ChatGPT.

  • Precio: gratis; Plus $20/mes ($16.67 anual).
  • Debilidad explotable: es librería de plantillas, no “optimiza tu texto”; muy ligado a ChatGPT.
PromptPerfect (Jina) — optimizador web

Optimizador de prompts basado en web.

  • Precio: $19.99/mes; Pro Max $99.99/mes.
  • Debilidad explotable: no tiene extensión de navegador; precio alto.
Otros (Prompt Genie, TaoPrompt, PromptHero)

Asistentes y librerías de prompts; más orientados a plataformas web que a un flujo de un clic dentro de cualquier página.

Monetización

El modelo actual es BYOK (cada usuario pone su clave). Eso define dos caminos.

Recomendado para empezar. Bajo costo, sin backend, lanzamiento rápido.
  • Tú no pagas la IA (la paga el usuario con su clave) → costo de operación casi cero.
  • Gratis: optimización básica. Pro: historial con búsqueda, plantillas/estilos personalizados, comparación lado a lado, puntuación de calidad, sincronización.
  • Cobro: ExtensionPay (Chrome + Firefox) o licencias vía Gumroad/Stripe.
  • Precio: $3–6/mes o pago único $15–25.
Mayor ingreso potencial, mayor complejidad. Tú asumes el costo de API.
  • Tú provees la IA mediante un backend/proxy con tu clave.
  • Gratis: X optimizaciones/mes. Pro: ilimitado.
  • Cero fricción para el usuario (no necesita conseguir clave), pero requiere backend, control de abuso y márgenes. Es el modelo de Promptimize ($12/mes).
  • Team/Empresa: varios asientos.
  • Lifetime para early adopters: ingreso inicial.
  • Cuña de mercado hispano: marketing enfocado a un segmento poco atendido.
Recomendación: lanzar con Camino A (freemium BYOK) — rápido, barato y sin infraestructura — y, si crece la demanda, añadir un tier “gestionado” (Camino B) como conveniencia premium.

Publicación en tiendas

Proceso para Firefox (principal) y Chrome/Edge.

Publicar en AMO es gratis (sin cuota de registro ni por extensión).
Crea una cuenta gratuita de Mozilla.
Developer Hub → Submit a New Add-on; sube el .zip.
Elige distribución “On this site” (listada) o autodistribución.
Revisión automática + posible revisión manual (por permisos sensibles). El código no está minificado, así que no deberían pedir build especial.
AMO firma el complemento. La instalación permanente requiere esa firma (la carga temporal es solo para pruebas).
Requiere una cuota de registro de $5 USD, única (cubre hasta 20 extensiones).
Regístrate en el Developer Dashboard y paga los $5.
Empaqueta un .zip con el manifest.json en la raíz.
Prepara recursos: icono 128px, 1–5 capturas, descripción, categoría, política de privacidad (URL obligatoria) y justificación de permisos.
Sube, completa la ficha y envía a revisión (de días a un par de semanas).
Pendiente antes de publicar: redactar una política de privacidad (ambas tiendas la exigen): la clave y los ajustes se guardan localmente; el texto seleccionado se envía solo al proveedor elegido; no hay servidores propios.
